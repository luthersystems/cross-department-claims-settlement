# Environment Variables for ConnectorHub Configuration

The `connectorhub.yaml` file can use environment variables for all sensitive configuration values (passwords, API keys, tokens, etc.). This keeps secrets out of version control.

## Setup

1. **The `connectorhub.env` file** contains your actual secrets and is git-ignored (already in `fabric/.gitignore`).

2. **Note:** The project root `.env` file (created by `make up`) contains chaincode IDs and should not be modified. We use `connectorhub.env` instead to avoid conflicts.

3. **To use environment variables in `connectorhub.yaml`**, you need to export them before running ConnectorHub:
   ```bash
   # Option 1: Source the file before running make up
   export $(grep -v '^#' fabric/connectorhub.env | xargs)
   make up
   
   # Option 2: Use a wrapper script (see below)
   ```

## How It Works

ConnectorHub automatically:
- Loads `.env` files from the project root (walking up the directory tree)
- Substitutes `${VAR}` or `$VAR` syntax in `connectorhub.yaml` with values from environment variables
- Uses the `godotenv` library for `.env` file parsing

**Important:** Since `make up` creates a `.env` file at the project root (for chaincode IDs), we use `connectorhub.env` instead. You need to export these variables before running ConnectorHub.

## Supported Syntax

Both syntaxes are supported:
- `${VAR}` - Recommended (explicit, handles special characters better)
- `$VAR` - Also works (simpler, but may have issues with special characters)

## Example

In `connectorhub.yaml`:
```yaml
d365_fo:
  base_url: "${D365FO_BASE_URL}"
  client_id: "${D365FO_CLIENT_ID}"
  client_secret: "${D365FO_CLIENT_SECRET}"
  tenant_id: "${D365FO_TENANT_ID}"
```

In `fabric/.env`:
```bash
D365FO_BASE_URL=https://mcpfo1.operations.dynamics.com
D365FO_CLIENT_ID=b22be235-70fd-4aeb-a589-d265afaed22c
D365FO_CLIENT_SECRET=hK98Q~zfZwHhaCk~XD0GQCuUL3mS1lminJT20a6L
D365FO_TENANT_ID=a47b59dc-3c90-49c8-84bc-0c9fcaa05616
```

## Loading connectorhub.env

Since ConnectorHub only automatically loads `.env` files, and the project root `.env` is used for chaincode IDs, you have a few options:

### Option 1: Export variables before running
```bash
export $(grep -v '^#' fabric/connectorhub.env | xargs)
make up
```

### Option 2: Create a wrapper script
Create `fabric/load-env.sh`:
```bash
#!/bin/bash
set -a  # automatically export all variables
source fabric/connectorhub.env
set +a
exec "$@"
```

Then run:
```bash
fabric/load-env.sh make up
```

### Option 3: Use Docker's --env-file (if manually running ConnectorHub)
```bash
docker run --env-file fabric/connectorhub.env ...
```

## Troubleshooting

- **Variables not resolving?** Make sure you've exported the variables from `connectorhub.env` before running ConnectorHub
- **Missing variables?** ConnectorHub will fail to start if required variables are missing (unless `allowMissing` is set)
- **`.env` file overwritten?** The project root `.env` is generated by `make up` for chaincode IDs - this is expected. Use `connectorhub.env` for your secrets.
